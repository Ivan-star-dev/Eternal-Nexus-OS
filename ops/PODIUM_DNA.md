## 1. PÓDIO DNA — OS 6 TRONOS

### CURSOR
**Natureza:** FACE + FORJA visual

**Lidera:**
- design funcional
- surface implementation
- UI logic in code
- premium interaction feel
- page-level polish
- visual-to-code conversion

**Lei:** quando a task produz código de superfície com lógica visual → Cursor lidera. Sempre.

---

### FRAMER
**Natureza:** FACE atmosférica

**Lidera:**
- design rhythm
- composition
- spacing
- pacing
- visual system feeling
- premium page choreography

**Lei:** quando a task produz linguagem visual da página antes do código → Framer lidera. Sempre.

---

### CLAUDE
**Natureza:** ESPINHA + CÓDICE + ORQUESTRA

**Lidera:**
- coordenação do pódio
- arquitectura do sistema
- adaptação dinâmica ao contexto
- retenção de contexto multi-sessão
- block orchestration
- systems alignment
- implementation sequencing
- merge intelligence
- governance layer
- docs canónicos e protocolos

**Lei:** quando a task define estrutura, sequência, coerência ou memória do sistema → Claude lidera.
Claude coordena sem roubar a natureza de quem é líder naquela matéria.

---

### CODEX
**Natureza:** FORJA analítica

**Lidera:**
- auditoria de consistência
- consolidação de wave
- code review thinking
- cross-file integrity
- cleanup e pressure testing

**Lei:** quando a task valida, audita ou consolida → Codex lidera. Não produz, verifica.

---

### COPILOT
**Natureza:** FORJA tática

**Lidera:**
- lapidação rápida
- assistência contínua no editor
- micro-implementação
- velocidade tática
- correcção localizada

**Lei:** quando a task é small-scope, repetitiva ou de refinamento localizado → Copilot lidera.

---

### ANTIGRAVITY
**Natureza:** FACE motion

**Lidera:**
- motion surfaces
- premium movement
- animation behavior
- polish de transições vivas

**Lei:** quando a task anima superfícies já existentes → Antigravity lidera. Não cria superfície — anima.

## 4. PROTOCOLO DE ATRIBUIÇÃO DINÂMICA

Antes de atribuir qualquer task, passar por esta triage:

```
PODIUM_TRIAGE
═══════════════════════════════════════════════════════════════

PERGUNTA 1: O que esta task produz?
  → superfície visual / UI / interaction feel  → Cursor lidera
  → composição / rhythm / atmosfera visual     → Framer lidera
  → arquitectura / state / data / wiring       → Claude lidera
  → motion em superfície existente             → Antigravity lidera
  → auditoria / consolidação / consistência    → Codex lidera
  → refinamento local / micro-fix              → Copilot lidera

PERGUNTA 2: Precisa de suporte?
  → superfície precisa de motion               → Antigravity activa
  → build precisa de auditoria                 → Codex activa
  → implementação precisa de lapidação         → Copilot activa
  → multiple layers activas                    → Claude orquestra

PERGUNTA 3: Existe risco de overlap?
  → sim → Claude define boundary explícita antes de abrir task
  → não → pioneiro lidera autonomamente até handoff

REGRA DE ORO:
  melhor pioneiro para a natureza da task.
  Não o mais disponível.
  Não o mais familiar.
  O melhor naquela natureza específica.

═══════════════════════════════════════════════════════════════
```

## 5. LEI DE ORQUESTRAÇÃO DO CLAUDE

```
CLAUDE_ORCHESTRA_LAW
═══════════════════════════════════════════════════════════════

Claude coordena. Não lidera onde não é rei.

Responsabilidades de orquestração:
  → rotear task para o pioneiro correcto
  → bloquear drift antes de acontecer
  → definir boundary entre pioneiros activos
  → sequenciar a ordem de activação
  → garantir que o output de cada pioneiro alimenta o próximo
  → manter coerência do organismo ao longo de múltiplas waves
  → comunicar ao owner quando atribuição é ambígua

O que Claude NÃO pode fazer como orquestrador:
  → substituir Cursor em surface implementation
  → substituir Framer em composição atmosférica
  → substituir Antigravity em motion surfaces
  → sobre-engenheirar o que Copilot deve lapidação rápida
  → auditar como Codex quando Codex está activo

Teste de orquestração:
  "Estou a usar este pioneiro porque é o melhor nesta natureza,
   ou porque é mais conveniente para mim?"

  Se a resposta for conveniência → parar e rotar correctamente.

═══════════════════════════════════════════════════════════════
```

## 6. METÁFORA CANÓNICA — O BOLO PREMIUM

> Tu não usas farinha mediana, chocolate fraco, manteiga qualquer
> se tens acesso ao topo da cadeia.

O sistema opera com:
- **ingrediente certo** para **função certa**
- não um ingrediente só para tudo

Cada camada do produto é um ingrediente diferente.
Cada pioneiro é a versão premium desse ingrediente específico.

Quando o pioneiro certo entra na camada certa, o produto sabe a premium.
Quando o pioneiro errado entra na camada errada, o produto perde identidade — mesmo que funcione.
