## SYSTEM_FACE_CANON

```
SYSTEM_FACE_CANON ══════════════════════════════════════════════════════════

O_QUE_E:
  A face do sistema operativo — o motor de governança, a sala de controlo
  viva da máquina. É o Eternal Nexus OS como ele mesmo. O lugar onde os
  pioneiros trabalham, os protocolos residem, os dados fluem.
  Não é um produto para o público. É o cockpit soberano.
  Referência de sensação: Bloomberg Terminal · Claude Code · ESA Mission Control.
  Calmo. Técnico. Elegante. Vivo.

FUNCAO:
  → Exibir o estado real do sistema em tempo contínuo
  → Comandar e monitorizar os pioneiros (BASTION, LIVE_STATE, NEXUS)
  → Navegar o arquivo de protocolos com precisão cirúrgica
  → Registar evidência e handoffs sem ornamento
  → Servir como interface entre o owner e o OS vivo

EMOCAO:
  → Confiança silenciosa — tudo está sob controlo, mesmo sem mostrar esforço
  → Autoridade tranquila — não precisa provar nada; o sistema já prova
  → Concentração profunda — como um cockpit antes do lançamento
  → Inteligência madura — não juvenil, não ansioso, não efusivo
  → Reverência técnica — respeito pela precisão, não pelo espectáculo

VISUAL_LANGUAGE:
  → Fundo deep navy #060c14 — permanente, nunca branco, nunca cinzento
  → Grid técnico subtil no background (opacity 0.04–0.06) — substrato sempre presente
  → Tipografia dominante: JetBrains Mono para dados/comandos + Syne 300–400 para UI
  → Cor operacional: teal #206358 como sinal de "ativo" / "vivo" / "conectado"
  → Gold #c9870f só para marcadores de autoridade (owner, decisão soberana, gate aberto)
  → Animação: mecânica e precisa — como cursor a piscar, não como orb a flutuar
  → Superfícies glass mínimas — apenas onde necessário para separar camadas
  → Densidade de informação: média-alta — este não é um hero vazio
  → Cursor/prompt visual: linha de comando visível; pode haver prompt símbolo ( › ou ▸ )

O_QUE_HERDA_DA_MAE:
  → Paleta core: navy + teal + gold (BRAND_MOTHER_SYSTEM.md secção 2.1)
  → Glass anatomy: atlas-glass-fill 0.04, blur 16px (quando aplicado)
  → Motion easing: [0.22,1,0.36,1] — mesmo com carácter mais mecânico
  → Anti-codes: calmo · sem urgência artificial · sem saturação
  → Regra de silêncio: mesmo terminal respira — não comprime tudo
  → Separadores e bordas da mesma lei (border-white/5–10)

O_QUE_NUNCA_PODE_VIRAR:
  ✗ Landing page de produto — sem hero com tagline motivacional
  ✗ Dashboard SaaS genérico — sem cards coloridos por categorias
  ✗ Hacker estético — sem verde matrix, sem texto a cair, sem ruído propositado
  ✗ Cyberpunk — sem neon excessivo, sem distortion, sem glitch decorativo
  ✗ Interface de consumer app — sem onboarding friendly, sem ilustrações
  ✗ Portfólio de agência — sem showcase de projetos com thumbnails coloridas
  ✗ Urgente ou agitado — sem badges pulsantes vermelhos sem razão real

══════════════════════════════════════════════════════════════════════════════
```

## PRODUCT_FACE_CANON

```
PRODUCT_FACE_CANON ══════════════════════════════════════════════════════════

O_QUE_E:
  A face pública do Heaven Lab — o produto-manifesto que o mundo vê.
  É onde a visão se torna experiência real. O lugar da presença,
  da trindade (Heaven Lab / Bridge Nova / Nexus Cria), do living globe,
  da câmara orbital sagrada. É o flagship público do sistema.
  Referência de sensação: Observatório científico premium · Apple WWDC · ESA probe landing.
  Espacial. Contemplativo. Atmosférico. Inesquecível.

FUNCAO:
  → Apresentar o Heaven Lab como categoria nova (não produto, não serviço)
  → Comunicar a trindade (sistema + produto + legado) de forma experiencial
  → Passar o Heaven Lab Test: visitante diz "isto não é normal"
  → Ser a primeira prova pública de que o sistema existe e é coerente
  → Criar dwell time: visitante fica, rola devagar, volta

EMOCAO:
  → Maravilha — algo que não se esperava encontrar na web
  → Calma planetária — como observar a Terra do espaço
  → Curiosidade sem pressa — a experiência convida a explorar, não a converter
  → Pertença — quem fica sente que encontrou algo que já procurava
  → Respeito pela inteligência do visitante — nunca infantilizar, nunca simplificar demais

VISUAL_LANGUAGE:
  → Living globe 3D como centro — presença viva, não ilustração
  → Tipografia fina e editorial: Cormorant Garamond para textos-chave + Syne 300 para UI
  → Espaço negativo generoso: mínimo 40% no hero — o silêncio é elemento activo
  → Câmara orbital: glass subtil ao redor do globo, profundidade z-layer controlada
  → Paleta: navy profundo + gold para pontos de atracção + teal para dados vivos
  → Aurora / orbs atmosféricos: presença suave, nunca distracção
  → Motion: orbital lento (8000ms), entrada suave (700ms), stagger nos filhos
  → Trindade orbital: 3 nós em 120°, aparecem em stagger, hover expande identidade

O_QUE_HERDA_DA_MAE:
  → Paleta completa da mãe (inclusive morabeza em camadas humanas/founder)
  → Glass anatomy com blur-md 16px como lei
  → Motion easing [0.22,1,0.36,1] para todas as entradas
  → Syne para labels e navegação; Cormorant para editorial
  → Regra de silêncio: hero com 40% negative space mínimo
  → Heaven Lab Test como critério de validação (BRAND_MOTHER_SYSTEM.md)
  → Ícones Lucide React com tom paper-dim

O_QUE_NUNCA_PODE_VIRAR:
  ✗ Terminal puro — sem prompt, sem cursor piscante, sem densidade técnica
  ✗ Dashboard de dados — sem tabelas, sem gráficos no hero
  ✗ Site de startup com pitch — sem "we help X do Y", sem social proof badges
  ✗ Portfólio — sem grid de projetos com thumbnails no fold acima
  ✗ Marketing pesado — sem CTAs agressivos, sem escassez artificial
  ✗ Interface plana — sem profundidade, sem atmosfera, sem presença 3D/motion
  ✗ Familiar / déjà vu — o visitante nunca deve sentir que já viu isto

══════════════════════════════════════════════════════════════════════════════
```

## ECOSYSTEM_FACE_CANON

```
ECOSYSTEM_FACE_CANON ════════════════════════════════════════════════════════

O_QUE_E:
  A membrana conectiva viva — a camada que liga sistema + produto + corporação
  + memória + didáctica. É a face do ecossistema em movimento.
  Não governa como o sistema. Não impressiona como o produto.
  Conecta, orienta, cataloga, revela ligações entre partes.
  Referência de sensação: Wikipedia de um laboratório · mapa conceptual vivo · arquivo inteligente.
  Estruturado. Navegável. Conectado. Silencioso.

FUNCAO:
  → Ligar os artefactos canônicos entre si de forma navegável
  → Exibir o ecossistema como mapa vivo (não lista estática)
  → Servir de ponte entre a profundidade técnica do sistema e a experiência do produto
  → Guardar e apresentar memória, didáctica, protocolos, aprendizagens
  → Permitir exploração não-linear do sistema por agentes e humanos

EMOCAO:
  → Orientação — o visitante sabe onde está e para onde pode ir
  → Descoberta calma — cada ligação revela algo novo sem surpresa excessiva
  → Confiança estrutural — o ecossistema parece completo, coerente, vivo
  → Profundidade silenciosa — quanto mais se explora, mais se encontra
  → Nem frio nem quente — equilíbrio entre o técnico e o humano

VISUAL_LANGUAGE:
  → Tipografia: Syne 300–500 dominante — leve, estruturada, neutra
  → JetBrains Mono para metadados (versão, data, ID, status)
  → Cormorant apenas em citações ou frases filosóficas de ligação
  → Densidade de informação: média — mais que o produto, menos que o sistema
  → Superfícies glass mais pronunciadas que no produto (separação de camadas)
  → Grid visual presente mas mais suave que no sistema (opacity 0.03)
  → Cor: navy base + paper-dim para texto neutro + teal para links/conexões vivas
  → Gold com contenção — apenas para pontos canônicos ou decisões soberanas
  → Motion: transições de 400ms — mais rápido que o produto, mais suave que o sistema

O_QUE_HERDA_DA_MAE:
  → Paleta core: navy + teal + paper (gold com contenção)
  → Glass anatomy da mãe como separador de camadas
  → Motion easing padrão [0.22,1,0.36,1]
  → Syne como família principal (UI + navegação)
  → Regra de silêncio: nem mesmo o ecossistema comprime informação
  → Ícones Lucide React com tom teal (conexão activa) ou paper-dim (inativo)

O_QUE_NUNCA_PODE_VIRAR:
  ✗ Dashboard corporativo — sem cards com KPIs coloridos por categoria
  ✗ Landing page — sem hero motivacional, sem CTA no fold
  ✗ Terminal puro — sem comando line como interface principal
  ✗ Produto experiencial — sem globe, sem orbs dominantes, sem motion atmosférico pesado
  ✗ Wikipedia plain — sem tipografia sem alma, sem presença visual
  ✗ Mapa genérico — sem nodes coloridos aleatoriamente, sem linha temporal de startup
  ✗ Mistura de faces — não imitar sistema no ecossistema, não imitar produto no ecossistema

══════════════════════════════════════════════════════════════════════════════
```
