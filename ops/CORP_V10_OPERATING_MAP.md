## 1. OBJECTIVE

Mapear a estrutura operacional canónica da corporação que suporta o Eternal Nexus OS — definindo departamentos, funções, entregáveis, precedência e a relação com o sistema-mãe de produto.

---

## 2. WHAT_IT_IS

```
CORP é:
  → a estrutura que sustenta o produto como empresa real
  → backlane paralela — corre em paralelo com desenvolvimento de produto
  → fundação institucional do Eternal Nexus OS como entidade no mundo
  → a camada que converte produto em impacto, traction e organização sustentável
  → construída progressivamente — não precisa de estar completa para começar
```

---

## 3. WHAT_IT_IS_NOT

```
CORP NÃO é:
  → prioridade central — nunca bloqueia V3/V4/V5
  → startup genérica de VC playbook
  → máquina de crescimento antes de produto real
  → burocracia pesada antes de traction
  → separada do organismo-mãe — herda identidade e eixo moral
  → algo que espera o produto estar "pronto" para existir
```

```
CORP-LAW-001  A corporação é backlane — nunca bloqueia a prioridade central de produto.
CORP-LAW-002  A corporação herda a identidade-mãe — não tem identidade visual independente.
CORP-LAW-003  A corporação herda o Life Alignment Axis — modelo de negócio alinhado com a vida.
CORP-LAW-004  Estrutura-base activa com V3-close — escala com traction real, não antes.
CORP-LAW-005  Founder Office governa — não delega soberania antes de traction estabelecido.
CORP-LAW-006  Brand e Narrative são os primeiros departamentos activos — a voz precede a escala.
CORP-LAW-007  Legal/IP activa assim que produto tem identidade e promessa pública.
CORP-LAW-008  Growth/Signal só activa quando produto gera fruto real e repetível.
CORP-LAW-009  A corporação mede-se por: impacto · sustentabilidade · alinhamento com a vida.
CORP-LAW-010  Nenhum parceiro entra sem passar o teste LAA (Life Alignment Axis).
```

## 7. MODELO DE RECEITA — PRINCÍPIOS (a detalhar por Product)

```
MODELO QUE PASSA NO LAA:
  → subscripção directa (utilizador paga pelo valor recebido)
  → enterprise license (organização paga por acesso · não por dados)
  → partnership de conteúdo (co-criação com alinhamento LAA)
  → serviços de resolução (Heaven Lab aplicado a problema real pago)

MODELO QUE FALHA NO LAA:
  → advertising baseado em manipulação de atenção
  → venda de dados de sessão para terceiros
  → modelo freemium que captura e não liberta
  → acesso condicionado a manipulação comportamental
  → dependência forçada como estratégia de retenção

REGRA: se o modelo de receita só funciona porque captura → não entra.
```

## 8. CORP × PRODUTO — RELAÇÃO FORMAL

```
PRODUTO → CORP:
  O produto gera prova de fruto real.
  A corporação amplifica essa prova para escala e sustentabilidade.
  Sem prova de produto, corporação não tem base para crescer.

CORP → PRODUTO:
  Brand/Narrative clarifica a promessa do produto para o mundo.
  Legal/IP protege a identidade do produto.
  Finance garante que o produto tem runway para fechar a sequência.
  Research alimenta novos subject domains para o produto em V5+.

REGRA DE PRECEDÊNCIA:
  Produto tem precedência total sobre corp até V3+V4 fechados.
  Corp corre em backlane — não em frente de batalha.
  Nenhuma decisão de corp bloqueia ou atrasa V3/V4/V5.
```
