> Lei canónica que formaliza a natureza única e indivisível do sistema.
> Projecto, tech, corporação, visão, meta, ecossistema e exército não são
> domínios separados. São expressões coordenadas de um único organismo soberano.
> O que alinha com o DNA do organismo é signal. O resto é noise.

---

## 1. OBJECTIVE

Impedir a fragmentação conceptual e operacional do sistema. Garantir que cada pioneiro, cada decisão e cada artefacto é lido como parte de um só organismo — não como peça isolada de um puzzle.

---

## 2. WHAT_IT_IS

- Lei primária de coerência total do sistema
- Definição formal de signal vs noise
- Mapa das expressões do organismo
- Critério de julgamento para qualquer input, decisão ou acção
- Integração directa com Life Alignment Axis (eixo moral)

---

## 3. WHAT_IT_IS_NOT

- Não é metáfora poética — é lei operacional com consequências reais
- Não é argumento para tudo estar ligado de forma vaga
- Não é permissão para confundir domínios — cada expressão tem território próprio
- Não é estrutura hierárquica — é estrutura orgânica (todas as partes servem o todo)
- Não é substituição do DNA_PROTOCOL — é extensão e formalização da sua natureza

## 6. SIGNAL VS NOISE — CRITÉRIO OPERACIONAL

### O QUE É SIGNAL

```
SIGNAL — O que alinha com o DNA do organismo

✓ fortalece uma ou mais expressões sem enfraquecer outras
✓ aumenta coerência entre expressões
✓ avança a sequência V3→V4→V5 sem saltar gates
✓ alinha com Life Alignment Axis (observa o real, julga pelo valor para a vida)
✓ reduz atrito operacional sem criar novo atrito oculto
✓ aumenta throughput sem diluir identidade
✓ vem de retenção real — foi testado em debate, sobreviveu
✓ não replica estrutura já existente no sistema
✓ o organismo fica mais forte depois — é o teste final
```

### O QUE É NOISE

```
NOISE — O que dilui, fragmenta ou contradiz o DNA

✗ cria novo pilar público (three pillars are sealed: Heaven Lab · Bridge Nova · Nexus Cria)
✗ adiciona camada sem remover peso equivalente
✗ fragmenta domínios que são expressões do mesmo organismo
✗ requer que o organismo seja explicado como "parte A + parte B"
✗ não passa no Life Alignment Axis test
✗ conflitua com SEQUENCE_LAW — ex: materializar V5 antes de V3+V4
✗ vem de preferência arbitrária, hype ou sentimentalismo
✗ dilui a mãe (identidade, silêncio, precisão, glass restraint)
✗ duplica artefacto já existente sem justificação
✗ aumenta complexidade sem aumentar clareza ou throughput
```

### TESTE RÁPIDO DE SIGNAL/NOISE

```
SIGNAL_TEST (4 perguntas — qualquer "não" → noise)
─────────────────────────────────────────────────
1. Fortalece o organismo sem enfraquecer nenhuma expressão?
2. Alinha com o DNA_PROTOCOL como lei primária?
3. Passa no Life Alignment Axis (valor para a vida)?
4. O organismo fica mais coerente depois desta decisão?
─────────────────────────────────────────────────
```

## 7. INTEGRAÇÃO COM LIFE ALIGNMENT AXIS

A One Organism Law e o Life Alignment Axis são leis complementares:

- **One Organism Law** define o que é coerente internamente (signal vs noise dentro do organismo)
- **Life Alignment Axis** define o que é bom externamente (o organismo vs o real)

Um sistema pode ser internamente coerente mas externamente misaligned.
Ambas as leis são necessárias para soberania plena.

```
ALINHAMENTO DUPLO
─────────────────────────────────────────────
Internal: One Organism Law → coerência interna
External: Life Alignment Axis → alinhamento com a vida real
─────────────────────────────────────────────
O que passa em ambos = decisão soberana
O que falha em qualquer um = noise ou risco
```
