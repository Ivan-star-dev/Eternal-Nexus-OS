## 1. OBJECTIVE

Formalizar a arquitectura do runtime soberano que interpreta o utilizador e a sessão e activa dinamicamente a constelação de capacidades necessária para materializar o melhor ambiente cognitivo dentro da identidade-mãe do sistema.

---

## 2. WHAT_IT_IS

```
SWMR é a camada de inteligência entre o utilizador e as capacidades do sistema.

Interpreta:
  → quem é a pessoa (perfil, histórico, nível, modo cognitivo)
  → o que quer fazer (subject, intenção, missão declarada ou inferida)
  → como aprende/cria/pensa (ritmo, profundidade, modo de resposta preferido)
  → o estado actual da sessão (fase, progressão, contexto aberto)

Activa:
  → a face certa da trindade (Heaven Lab · Bridge Nova · Nexus Cria)
  → o regime operativo adequado
  → a constelação de capacidades especializadas correcta
  → o ambiente visual e cognitivo alinhado com a mãe

Materializa:
  → laboratório → Heaven Lab (resolve · simula · decide)
  → escola → Bridge Nova (orienta · ensina · transfere)
  → estúdio → Nexus Cria (cria · produz · exporta)

Persiste:
  → memória de sessão
  → progressão
  → retorno mais rico
  → fruto tangível no fim
```

---

## 3. WHAT_IT_IS_NOT

- Wrapper de providers de AI (não é plugin-zoo com interface)
- Sistema de recomendação de ferramentas
- Dashboard configurável pelo utilizador
- Chatbot com personalidade
- Plataforma de AI generative genérica
- Lista de funcionalidades à escolha do user
- Sistema que o utilizador tem de "montar"

## 4. CORE_LAWS

```
SWMR-LAW-001  O ambiente adapta-se ao utilizador — o utilizador não monta o ambiente.
SWMR-LAW-002  A face activa é determinada pelo subject + intenção + perfil — não pela UI.
SWMR-LAW-003  A sessão não morre — tem ciclo completo: entrada → lock → progresso → fruto.
SWMR-LAW-004  Toda sessão deve gerar fruto real — sem fruto é falha de design do runtime.
SWMR-LAW-005  A costura é invisível — o utilizador vê um organismo, não os engines por baixo.
SWMR-LAW-006  A identidade-mãe é invariante — o ambiente muda, a atmosfera não.
SWMR-LAW-007  Memória activa: cada sessão sabe onde a anterior ficou.
SWMR-LAW-008  Profundidade progressiva: o sistema aprofunda à medida que o utilizador avança.
SWMR-LAW-009  O routing é silencioso — o utilizador não escolhe modelos ou engines.
SWMR-LAW-010  A agência amplifica-se — a pessoa sai mais capaz, não mais dependente.
```

## 6. SESSION LIFECYCLE — CICLO FORMAL

```
FASE 1 — ENTRADA
  Utilizador chega ao sistema.
  Sistema reconhece: novo? recorrente? context known?
  Se recorrente: carrega memória + progressão anterior.

FASE 2 — LOCK DE INTENÇÃO
  Sistema resolve intenção: declarada ou inferida.
  Confirma com utilizador se ambíguo (sem formulário · linguagem natural).
  Lock: subject + regime + face activa.

FASE 3 — MATERIALIZAÇÃO
  Face activa: laboratório / escola / estúdio.
  Ambiente visual e cognitivo: configurado dentro da identidade-mãe.
  Capacidades: constelação activa sem visibilidade de engines.

FASE 4 — APROFUNDAMENTO
  Sistema aprofunda progressivamente à medida que sessão avança.
  Profundidade cresce com: perguntas · clareza · dados emergentes.
  Sistema adapta: ritmo · formato · nível de detalhe.

FASE 5 — FRUTO
  Toda sessão deve entregar fruto tangível:
    → clareza · output · artefacto · simulação · entendimento · decisão · progresso
  Se sessão não converge para fruto → sistema detecta e redirecciona.

FASE 6 — PERSISTÊNCIA
  Fruto registado.
  Memória actualizada.
  Progressão guardada.
  Sessão futura começa onde esta ficou.
```

## 10. ANTI_DRIFT_RULES

```
SWMR-DRIFT-1: Runtime reduzido a "lista de providers" → não. É camada de inteligência.
SWMR-DRIFT-2: Utilizador escolhe engines/modelos → não. Routing é invisível.
SWMR-DRIFT-3: Sessão sem fruto aceite → não. Falha de design · redesenhar.
SWMR-DRIFT-4: Identidade-mãe alterada pelo regime → não. Layer 7 é invariante.
SWMR-DRIFT-5: SWMR implementado antes de V3 fechar → não. Sequência é lei.
SWMR-DRIFT-6: Ambiente configurado pelo utilizador manualmente → não. Sistema configura.
SWMR-DRIFT-7: Memória descartada entre sessões → não. Persistência é requisito.
SWMR-DRIFT-8: Face escolhida por clique de menu → não. Face activada por intenção.
```
