1. **PRs empilhados no branch canônico** — o branch de trabalho ativo é `claude/expose-workspace-config-yt4Km`, não `main`. PRs que apontam para esse branch ficam visíveis no GitHub mas nunca chegam a `main`.

3. **PRs de relatório nunca fechados** — PRs informativos (como #73: triage report) são criados, lidos e esquecidos sem fechamento.

## CAUSA RAIZ SISTÊMICA

O acúmulo acontecerá novamente se:
- Sessões de agentes abrirem PRs novos em vez de atualizar os existentes ao falhar
- O branch canônico não for `main` (PRs em branches intermediários são invisíveis ao fluxo de merge)
- PRs informativos (relatórios, triagens) não forem fechados após leitura

**Solução preventiva:** ao iniciar uma sessão, verificar se já existe PR aberto para a mesma task antes de criar um novo.
