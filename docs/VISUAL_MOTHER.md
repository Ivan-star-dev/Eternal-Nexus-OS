## 1. VISUAL MOTHER — IDENTIDADE

### 1.1 — Nome

**Aether** — a direção visual-mãe da Eternal Nexus Corporation.

O nome Aether vem do éter: o meio invisível que conecta tudo. A espinha técnica chama-se Aether Spine (`DOC_VISUAL_SPINE_001.md`). A extensão tangível chama-se Codex Tangibilis (`MASTER_DOSSIER_SYSTEM.md`). Ambas são filhas desta mãe.

### 1.2 — Essência

Uma corporação que parece ter sido desenhada por um único ser com obsessão por coerência. Terminal AI nostálgico com calma computacional. GitHub clean encontra polimento Cursor/Framer. Manual técnico dos anos 70 redesenhado para 2026 com inteligência artificial disciplinada.

### 1.3 — Sensação dominante

| Atributo | Presença |
|---|---|
| **Calma computacional** | Paz mental visual em cada superfície — nenhum elemento compete por atenção |
| **Nostalgia técnica** | Sensação de artefato raro, terminal clássico, monospace com alma |
| **Nova era de AI** | Precisão, observabilidade, rastreabilidade como qualidades visuais — não como decoração |
| **Corporação viva** | Séria mas não morta; organizada mas não rígida; premium mas acessível |
| **Intemporal** | Não envelhece em 2 anos; não persegue tendência; respira entre passado e futuro |

### 1.4 — O que pertence a Aether

```
✓ Branco vinil esfumaçado como campo principal
✓ Calor amarelado ultra-sutil (presença de corpo, não sépia)
✓ Tipografia mono/machine elegante como voz de sistema
✓ Linhas finas, hairline, funcionais
✓ Verde terminal como vetor semântico forte
✓ Colunas e cartões flutuantes — sensação de almofada de nuvem
✓ Contraste calmo — alto o suficiente para ler, baixo o suficiente para não agredir
✓ Superfícies de vidro opaco com profundidade curta
✓ Ruído fino de textura (anti superfície chapada)
✓ Névoa suave como respiração visual
✓ Espaço em branco generoso como arma principal
✓ Ritmo visual: institucional → técnico → autoral
```

### 1.5 — O que não pertence a Aether

```
✗ Sci-fi bagunçado (neon, glitch, hologramas decorativos)
✗ Futurismo genérico (gradientes violeta-azul, glow excessivo)
✗ Excesso robótico (estética fria sem alma humana)
✗ Maximismo visual (muitos elementos, muitas cores, muitas camadas)
✗ Branco hospitalar (branco puro sem corpo nem calor)
✗ Dark mode como default (escuridão não é a base; é exceção técnica controlada)
✗ Startup pitch aesthetic (cores primárias, ilustrações genéricas, entusiasmo visual)
✗ Ornamento sem função (bordas decorativas, ícones sem semântica, sombras pesadas)
✗ Tipografia display exuberante (a voz é mono e serifada, não expositiva)
```

### 1.6 — Como Aether se diferencia

| De... | Aether difere porque... |
|---|---|
| Sci-fi futurista genérico | Tem calma e nostalgia onde o genérico tem barulho e neon |
| Estética clean minimalista | Tem corpo (calor, textura, névoa) onde o minimalismo puro é chapado |
| Corporate boring | Tem alma humana e ritmo vivo onde o corporativo genérico é morto |
| Terminal retro puro | Tem polimento premium e composição moderna onde o retro é cru |
| Design system de startup | Tem intemporal sobriedade onde a startup tem entusiasmo temporal |

## 2. CORPORATE VISUAL FAMILY — RAMIFICAÇÃO

A mãe visual ramifica-se em 8 formatos-filho. Cada filho herda a essência e adapta para o seu contexto.

### 2.1 — Mapa de família

```
AETHER (mãe visual)
│
├── F1  INTERNAL OPS          operações internas, FOL, NLF, BASTION
├── F2  PROTOCOL              protocolos soberanos, leis, constituição
├── F3  HANDOFF               handoffs, evidence blocks, canalização
├── F4  REPORT                relatórios, consolidações, Tribunal
├── F5  ENGINEERING           GitHub, CI, README, instruções de agentes
├── F6  DOSSIER / TANGIBLE    Codex Tangibilis, artefatos imprimíveis
├── F7  LIVE METRICS          dashboards, terminais, estado vivo
└── F8  PRODUCT-FACING        site, UI, experiência pública
```

### 2.2 — Especificação por filho

| ID | Formato | Herda da mãe | Variação permitida | Tom |
|---|---|---|---|---|
| **F1** | Internal Ops | branco vinil, mono, hairline, verde terminal | densidade técnica máxima; leitura rápida | terminal operacional limpo |
| **F2** | Protocol | branco vinil, mono bold, linhas fortes | peso visual mais alto; autoridade legislativa | institucional soberano |
| **F3** | Handoff | branco vinil, tabelas fixas, semáforo verde/âmbar/vermelho | formato copiável; blocos em código | protocolar executivo |
| **F4** | Report | branco amarelado, progressão visual, seções claras | contraste alto em risco; cor por gravidade | corporativo analítico |
| **F5** | Engineering | branco limpo, mono puro, markdown clean | redução de atmosfera; máxima legibilidade técnica | GitHub clean operacional |
| **F6** | Dossier / Tangible | branco vinil esfumaçado, serif + mono, margens generosas | layout para papel; tipografia para impressão | manual premium intemporal |
| **F7** | Live Metrics | campo escuro controlado (exceção), verde terminal dominante, mono | inversão de campo permitida para legibilidade de terminal | terminal AI nostálgico |
| **F8** | Product-Facing | branco vinil, névoa, cartões flutuantes, narrativa visual | redução de densidade operacional; aumento de experiência | portal premium público |

### 2.3 — Regras de herança

| Regra | Aplicação |
|---|---|
| **Tipografia herdada** | Todos usam mono (JetBrains Mono, IBM Plex Mono) + serif (IBM Plex Serif) como base |
| **Paleta herdada** | Todos partem do branco vinil `#FAFAF7` + verde `#00C853` + ink `#1A1A1A` |
| **Hairline herdada** | Linhas finas são o separador universal — nunca bordas grossas |
| **Semântica cromática herdada** | Verde/âmbar/vermelho mantém significado em todos os filhos |
| **Variação de tom** | Permitida dentro da mesma família — nunca fora |
| **Excepção F7** | Único filho com campo escuro como default (contexto de terminal/dashboard) |

## 3. SENSORIAL LANGUAGE — VOCABULÁRIO DE SUPERFÍCIE

### 3.1 — Campos

| Nome | Valor | Sensação |
|---|---|---|
| **Branco vinil esfumaçado** | `#FAFAF7` | O campo principal — não é branco puro, tem corpo e calor mínimo |
| **Branco amarelado intemporal** | `#F5F3EC` | Superfície premium com nostalgia — como papel de livro técnico raro |
| **Branco de névoa** | `#F0EDE6` | Campo arqueológico, mais quente, para contexto secundário |

### 3.2 — Tintas

| Nome | Valor | Sensação |
|---|---|---|
| **Preto grafite técnico** | `#1A1A1A` | Texto principal — não é preto absoluto, tem calor orgânico mínimo |
| **Cinza quente funcional** | `#4A4A46` | Texto auxiliar, legendas fortes — presente sem competir |
| **Cinza caption** | `#7A7A72` | Metadados, notas de rodapé — recede sem desaparecer |

### 3.3 — Estrutura

| Nome | Valor | Sensação |
|---|---|---|
| **Hairline quente** | `#D4D2CA` | Linha fina de separação — presente mas não agressiva |
| **Linha de estrutura** | `#8A8880` | Divisores principais, cabeçalhos — peso funcional |

### 3.4 — Sinais

| Nome | Valor | Sensação |
|---|---|---|
| **Verde terminal elegante** | `#00C853` | Vivo, confirmado, execução, protocolo — o vetor mais forte |
| **Âmbar protocolar** | `#F5A623` | Transição, pendente, atenção — calmo mas presente |
| **Vermelho disciplinado** | `#FF4F2E` | Bloqueio, falha, atrito real — nunca decorativo, sempre funcional |
| **Ouro de selo** | `#C8A96E` | Autenticidade, premium restrito — raramente usado |

### 3.5 — Atmosfera

| Elemento | Descrição |
|---|---|
| **Névoa** | Translúcida, branca ou cinza muito suave, para separar camadas e dar profundidade |
| **Ruído fino** | Textura de baixa amplitude para evitar superfície chapada — grain sutil |
| **Glow** | Halo curto, verde terminal ou branco, para focar — nunca decorativo |
| **Profundidade** | Conseguida por peso tipográfico e espaço — sombras apenas curtas e leves |
| **Flutuação** | Cartões e blocos parecem pairar — almofada de nuvem, não caixa pesada |

### 3.6 — Tipografia

| Uso | Família | Sensação |
|---|---|---|
| **Voz de sistema** | Mono (JetBrains Mono, IBM Plex Mono, Space Mono) | Machine type elegante — o som da corporação |
| **Voz narrativa** | Serif (IBM Plex Serif, Source Serif Pro) | Corpo e profundidade — autoridade editorial |
| **Voz de dados** | Mono regular | Legibilidade técnica absoluta — tabelas, métricas, código |
| **Voz de selo** | Mono light uppercase | Caption técnico — metadados, rodapés, timestamps |
| **Voz do founder** | Serif italic | Autoria distinguida — presença humana no sistema |

## 4. SEMÂNTICA CROMÁTICA — SISTEMA DE SIGNIFICADO

O significado das cores é universal em toda a corporação. Nunca muda de formato para formato.

| Cor | Significado | Quando aparece | Quando NÃO aparece |
|---|---|---|---|
| **Verde** `#00C853` | vivo / confirmado / saudável / execução fluindo / protocolo ativo | estados activos, confirmações, sinais de sistema, assinatura Cursor | decoração, fundo de bloco inteiro, bordas genéricas |
| **Âmbar** `#F5A623` | transição / pendente / atenção / decisão requerida | alertas suaves, estados intermediários, gates aguardando, revisão | urgência máxima (isso é vermelho), decoração |
| **Vermelho** `#FF4F2E` | bloqueio / falha / atrito real / criticidade | erros, bloqueios B-*, red lines, estados de falha confirmada | warnings suaves (isso é âmbar), decoração |
| **Cinza quente** `#4A4A46..#7A7A72` | base / suporte / neutralidade / recuo | texto auxiliar, metadados, legendas, contexto secundário | sinais de estado (esses são verde/âmbar/vermelho) |
| **Branco vinil** `#FAFAF7` | campo principal / ordem / paz / elegância / calma | fundo de tudo, campo dominante, espaço entre elementos | nunca eliminado; sempre presente como base |
| **Ouro** `#C8A96E` | autenticidade / selo / premium restrito | selos de versão, marcas de autenticidade, assinatura rara | uso frequente (ouro gasta-se visualmente; reservar) |

### 4.1 — Regra de semáforo operacional

```
  🟢 VERDE    = tudo OK / confirmado / ativo / continuar
  🟡 ÂMBAR    = atenção / pendente / decisão necessária / transição
  🔴 VERMELHO = parar / bloqueio / falha / atrito
```

Esta codificação é idêntica ao semáforo do NEXUS_OS (`docs/NEXUS_OS.md` §6) e ao sistema de handoff/canalização (`ops/OUTPUT_STANDARD.md`). Aether formaliza o que o protocolo já pratica.

## 5. CONSISTÊNCIA TOTAL — REGRA DE SANGUE

### 5.1 — Declaração

> Tudo o que exigir presença sensorial na Eternal Nexus Corporation nasce desta mãe.
> Output fora da família não é aceite.
> Visual com ruído excessivo não é aceite.
> Visual sem paz mental não é aceite.
> Visual genérico futurista não é aceite.
> A corporação inteira deve parecer desenhada pelo mesmo sangue visual.

### 5.2 — Regras de sangue

| Regra | Aplicação |
|---|---|
| **Teste de família** | Se um artefato for colocado ao lado de outro da mesma corporação e não parecerem irmãos → rejeitado |
| **Teste de 3 segundos** | Se um observador externo não reconhecer "Eternal Nexus" em 3 segundos → rejeitado |
| **Teste de paz** | Se olhar para o artefato causar ruído visual ou ansiedade → rejeitado |
| **Teste de intemporal** | Se o artefato parecer datado de uma tendência específica → rejeitado |
| **Teste de calor** | Se o artefato parecer hospitalar, frio ou sem corpo → rejeitado |
| **Teste de peso** | Se o artefato parecer fraco, genérico ou descartável → rejeitado |

### 5.3 — Gate de aceitação visual

Todo artefato sensorial passa por este gate antes de aceitação:

```
[ ] Usa campo branco vinil como base?
[ ] Tipografia é mono + serif da família?
[ ] Verde terminal usado com função semântica?
[ ] Linhas são hairline funcionais?
[ ] Espaço branco é generoso?
[ ] Passa no teste de 3 segundos?
[ ] Passa no teste de paz mental?
[ ] Não tem ornamento sem função?
[ ] Parece irmão dos outros artefatos?
```

## 6. APLICAÇÃO: ECOSSISTEMA → PRODUTO

### 6.1 — Ordem de ramificação

```
PRIMEIRO:  ecossistema interno (ops, handoffs, protocolos, docs)
DEPOIS:    produto (site, UI, interfaces públicas)
O DNA É O MESMO. MUDA A MANIFESTAÇÃO.
```

### 6.2 — Diferenças entre interno e público

| Dimensão | Interno (ecossistema) | Público (produto) |
|---|---|---|
| **Densidade** | Alta — rastreabilidade, comando, estado | Média — narrativa, clareza, experiência |
| **Jargão** | Permitido (handoff, BASTION, IGNIÇÃO, gates) | Eliminado (conceitos traduzidos para linguagem natural) |
| **Formato** | Blocos copiáveis, tabelas, semáforo | Cartões flutuantes, progressão visual, storytelling |
| **Atmosfera** | Funcional com elegância | Experiencial com substância |
| **Verde terminal** | Presente como sinal de estado | Presente como acentuação de sistema |
| **Assinatura de pioneiro** | Visível (AURA_CURSOR:, cor matriz) | Invisível (o visitante não sabe quem construiu o quê) |

### 6.3 — O que nunca muda entre interno e público

| Invariante | Regra |
|---|---|
| **Tipografia base** | Mono + serif — mesma família |
| **Campo dominante** | Branco vinil esfumaçado |
| **Verde terminal** | Mesmo hex, mesma função semântica |
| **Hairlines** | Mesma espessura, mesma função |
| **Calma visual** | Paz mental preservada em ambos |
| **Reconhecibilidade** | Irmãos — não primos distantes |
