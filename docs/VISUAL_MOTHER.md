## 1) Interface-mãe do sistema

### Nome canônico
**Aether Canon Terminal**

### Definição
O Aether Canon Terminal é a interface-mãe da corporação: um único organismo de leitura, execução e decisão que funde conforto cognitivo, governança operacional e presença visual premium.

Não é moodboard. Não é colagem de referências.  
É um protocolo visual funcional para trabalho real.

### Princípio-mãe
**Calma na superfície, precisão no núcleo.**

## 2) Composição estrutural (arquitetura de interface)

## 2.1 Estrutura de 6 camadas

| Camada | Função | Influência principal |
|---|---|---|
| `L1_Corporate_Header` | identidade institucional, fase, branch, executor | corporação viva |
| `L2_Context_Rail` | semáforo, gate, dependências, estado de continuidade | GitHub operacional |
| `L3_Long_Read_Core` | leitura longa, raciocínio, decisões e narrativa técnica | ChatGPT/Claude |
| `L4_Execution_Deck` | comandos, diffs, evidência, resultado objetivo | Codex/Cursor |
| `L5_Pioneer_Signature_Band` | assinatura semântica de pioneiro sem quebrar unidade | Visual Spine |
| `L6_Handoff_Chain_Block` | passagem de bastão, next actor, continuidade do loop | protocolo vivo |

## 2.2 Grid funcional
1. **Topo fixo curto** (identidade + estado do sistema)  
2. **Centro expansível** (leitura longa + execução técnica)  
3. **Rodapé operacional** (handoff + chain + evidence)

## 2.3 Regra de composição
- Priorizar linearidade de leitura.
- Evitar painéis paralelos excessivos.
- Toda seção deve responder: contexto, ação, prova, próximo elo.

## 3) Composição sensorial (presença, ritmo, conforto)

## 3.1 Clima sensorial obrigatório
- Paz mental (respiração visual)
- Corporação viva (autoridade sem rigidez fria)
- Terminal AI nostálgico (clareza textual, foco em conteúdo)
- Energia de execução quando necessário (não contínua)

## 3.2 Ritmo visual
- **Abertura:** limpa, institucional, baixa fricção
- **Corpo:** leitura longa com contraste estável
- **Pico:** execução técnica de alta clareza
- **Fecho:** handoff objetivo e continuidade

## 3.3 Matéria visual
- Branco premium como base estrutural
- Névoa curta e ruído fino controlado
- Bordas hairline e profundidade curta
- Glow mínimo e intencional

## 3.4 O que é proibido
- Futurismo genérico
- Robótica visual
- Neon caótico
- Vidro fake excessivo
- Glow ornamental

## 4) Fusão dos pioneiros sem ruptura de unidade

## 4.1 Matriz de convivência

| Origem | Estado incorporado no terminal | Limite de uso |
|---|---|---|
| Claude | abertura estratégica e coerência de direção | sem aquecer demais toda a tela |
| GitHub | rastreabilidade, diff, estado de execução | sem transformar tudo em painel denso |
| ChatGPT | leitura longa confortável e progressiva | sem perder objetividade operacional |
| Codex | energia técnica de execução e decisão binária | só em áreas de ação/alerta |
| Cursor | acabamento, precisão e microclareza | sem excesso de detalhe ornamental |
| Framer | presença, cadência e camada narrativa | sem dominar a operação com espetáculo |

## 4.2 Regra de unidade
Cada influência entra como **função de sistema**, nunca como estilo isolado.  
Se um traço visual não melhora leitura, decisão ou continuidade, ele sai.

## 5) Blueprint aplicável ao sistema real

## 5.1 Mapeamento direto

| Artefato real | Como aplicar o Aether Canon Terminal |
|---|---|
| Protocolos (`ops/`) | Header corporativo + contexto rail + chain block |
| Prompts | abertura limpa + objetivo + limite + próximo ator |
| Handoffs | bloco operacional com assinatura de executor e evidence forte |
| Relatórios | corpo long-read + deck de decisões + síntese executiva |
| Docs canônicos | leitura longa premium com hierarquia rígida |
| Interfaces internas futuras | mesmas 6 camadas com estados vivos |

## 5.2 Critério de aceite
Uma implementação está correta quando o leitor consegue, em sequência:
1) entender o contexto  
2) localizar a ação  
3) validar a prova  
4) saber quem entra a seguir
