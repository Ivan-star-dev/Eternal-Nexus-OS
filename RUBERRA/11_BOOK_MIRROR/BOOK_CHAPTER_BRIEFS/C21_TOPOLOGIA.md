## PERGUNTA QUE ESTE CAPÍTULO RESOLVE
Como o sistema está fisicamente organizado — nodos, conexões, distâncias?

## CONTEÚDO PRINCIPAL
- Topologia do sistema: nodos, edges, clusters
- Anatomia interna: o que está dentro de cada nodo
- Distâncias canônicas entre componentes
- Pontos de falha e redundâncias

## HIERARQUIA VISUAL
MACRO → network topology map full spread
MESO → nodos com tipos e funções
MICRO → edges com labels de protocolo

## ELEMENTOS VISUAIS
- Network diagram estilo força-directed
- Nodos maiores para componentes de maior centralidade
- Edges com espessura proporcional ao volume de fluxo
- Clusters identificados por cor

## TOM
Técnico. A topologia não precisa de palavras para ser compreendida.
