## PERGUNTA QUE ESTE CAPÍTULO RESOLVE
Quais os padrões concretos que tudo deve seguir?

## CONTEÚDO PRINCIPAL
- Naming standards: maiúsculas canônicas, monospace para IDs
- Visual grammar: gold/teal/black, serif/mono/sans
- Template structures: HANDOFF · BASTION task block · SITUATION REPORT
- O que acontece quando um standard é violado

## HIERARQUIA VISUAL
MACRO → "STANDARDS" como título dominante
MESO → categoria de standard com regras
MICRO → exemplos corretos vs incorretos

## ELEMENTOS VISUAIS
- Standard cards por categoria
- Exemplo visual: correto (verde) vs incorreto (vermelho)
- Template showcase: um exemplo de cada template canônico

## TOM
Claro e prescritivo. Não há ambiguidade — ou cumpre o standard ou não cumpre.
