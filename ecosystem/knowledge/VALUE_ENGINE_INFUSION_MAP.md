## LAB RECEIVES (Domain F primary + Domain A/C support)

### Primary: Testing and Validation Frameworks

```
VALIDATION FRAMEWORKS (ready for TestBay):

V-01: OFFER VALIDATION FRAME
  Hypothesis: [target person] has [problem] so acute that they will exchange [price] to solve it
  Test: direct conversation with 5+ qualified prospects
  Pass: 3+ signal genuine purchase intent without prompting
  Fail: need to explain the problem before they recognize it

V-02: POSITIONING VALIDATION FRAME
  Hypothesis: this message lands with [target] and clearly differentiates from [alternative]
  Test: show message to 5 ideal prospects — ask them to describe what it offers in their own words
  Pass: description matches intent without prompting
  Fail: confusion, misattribution, or need for clarification

V-03: PRICING VALIDATION FRAME
  Hypothesis: [price] reflects perceived value accurately for [segment]
  Test: Van Westendorp price sensitivity test or anchor/decoy test
  Pass: price sits in the acceptable zone without triggering high-cost rejection
  Fail: universal price objection independent of product quality

V-04: CHANNEL VALIDATION FRAME
  Hypothesis: [channel] reaches [ideal person] with sufficient density to be efficient
  Test: 30-day signal test — post/participate in channel, measure qualified responses per hour invested
  Pass: qualified signal rate exceeds threshold (defined per channel type)
  Fail: volume without qualification signal — wrong channel

V-05: TRACTION HYPOTHESIS FRAME
  Hypothesis: if [X] is true, we will observe [Y] within [Z] time
  Test: define the minimum observable signal that would confirm X
  Pass: signal observed within timeframe
  Fail: signal absent — X is not yet true, or Y is wrong proxy

V-06: PROTOTYPE VALIDATION FRAME
  Hypothesis: this minimum version of the product/offer creates the claimed value
  Test: deliver to 3+ early users with no hand-holding
  Pass: users achieve the promised outcome without additional support
  Fail: users achieve outcome only with significant creator intervention — product gap

V-07: BENCHMARK COMPARISON FRAME
  Hypothesis: this system performs better than [reference standard] on [specific dimension]
  Test: define dimension precisely, compare outputs directly
  Pass: measurable superiority on defined dimension
  Fail: parity or inferiority — does not yet surpass the benchmark
```

### Lab Product Scope (what Lab creates as artifacts)
- `hypothesis` — falsifiable claim about a value, offer, market, or system
- `experiment` — structured test with defined conditions and success criteria
- `evidence` — observed fact, stripped of interpretation
- Result chains: hypothesis → experiment → evidence → conclusion → law or rejection

### Lab Forbidden Content
- Persuasion mechanics (that's Creation)
- Teaching curricula (that's School)
- Strategy doctrine (that's School)
- Offer design (that's Creation)

## CREATION RECEIVES (Domains A, B, C, D primary)

### Product Creation Frameworks

```
PRODUCT ARCHITECTURE MAPS (ready for Creation Lab quick-create):

P-01: DIGITAL PRODUCT SYSTEM MAP
  Categories:
  → Info products: book, guide, report, research doc, framework doc
  → Teaching products: course, cohort, curriculum, workshop
  → Tool products: SaaS, template, toolkit, plugin, workflow pack
  → Service products: consulting, done-for-you, advisory, retainer
  → Authority products: membership, newsletter, private community, mastermind

  Selection criteria:
  → proximity to proven problem (closer = faster validation)
  → leverage ratio (how much of creator presence is required per delivery)
  → ascension compatibility (does this product lead naturally to a higher offer?)

P-02: OFFER DESIGN BLUEPRINT
  Structure:
  → Promise: what is the specific outcome? (measurable, time-bounded where possible)
  → Mechanism: why does this work when others fail?
  → Proof: what evidence exists that this produces the promised outcome?
  → Price anchor: what is the reference frame that makes this price feel right?
  → Risk reversal: what makes saying yes feel safe?
  → Boundary: what is explicitly excluded? (defines the offer, prevents scope creep)

  Quality test:
  → A skeptical, intelligent competitor could read this and not find a logical hole

P-03: LEVERAGE SYSTEM MAP
  Leverage types (ordered by scale potential):
  → 1→many teaching: one input, unlimited learners
  → 1→many media: one piece of content, unlimited views
  → 1→many tools: one build, unlimited users
  → 1→many frameworks: one model, unlimited applications
  → Network leverage: each user creates value for other users

  Selection criteria:
  → What is the creator's natural 1→many mode?
  → What leverage type is most validated in this domain?

P-04: AUTHORITY SYSTEM BLUEPRINT
  Components:
  → Intellectual territory: what domain does this entity own a distinct perspective on?
  → Named framework: what proprietary model or method is associated with this entity?
  → Consistent signal: what is the recurring output that accumulates trust?
  → Social proof chain: what external signals confirm authority? (testimonials, citations, track record)
  → Content spine: what is the central idea that everything else connects to?

P-05: VALUE LADDER ARCHITECTURE
  Levels:
  → Free tier: trust-building, problem-awareness, proof of value
  → Entry tier: low-commitment exchange, first transformation
  → Core tier: primary offer, full promise delivery
  → Premium tier: done-with-you, accelerated, closer access
  → Sovereign tier: advisory, custom, intimate — by invitation or application

  Law: each level must create genuine transformation, not just upsell to the next level.
  Continuity law: each level must create incompleteness that naturally pulls toward the next.
```

### Monetization Frameworks

```
M-01: PRICING LOGIC MAP
  Pricing as positioning signal:
  → Luxury positioning: price above market to signal category leadership
  → Premium positioning: price above average to signal quality above average
  → Value positioning: price at market with superior quality signal
  → Access positioning: price below market to capture volume, upsell later

  Pricing dimensions:
  → Cost-floor (minimum viable sustainability)
  → Perceived value ceiling (maximum credibility threshold)
  → Market anchor (what is the reference price in the buyer's mind?)
  → Competitor displacement (is this price point tactically superior?)

M-02: CONTINUITY MONETIZATION MAP
  Continuity types:
  → Subscription to access (membership, newsletter, platform)
  → Subscription to service (retainer, ongoing advisory)
  → Subscription to transformation (cohort, program, ongoing learning)
  → Subscription to community (network, peer group, mastermind)

  Continuity law: retention is earned by continuous delivery of the promised value,
  not by friction, lock-in, or forgetting to cancel.

M-03: TRUST-BASED SELLING MAP
  Trust funnel:
  → Awareness (they know the entity exists)
  → Problem recognition (they recognize their own problem through your framing)
  → Solution match (they believe this entity's approach matches their problem)
  → Proof acceptance (they trust that it works for people like them)
  → Risk acceptance (they believe the downside of buying is acceptable)
  → Exchange (they pay)

  Law: skipping steps doesn't accelerate the funnel — it creates refund requests.
```

### Creation Artifact Scope
- `plan` — execution map, offer blueprint, product architecture
- `research` — market intelligence, competitor analysis, audience insight
- `draft` — offer copy, curriculum structure, product outline
- `synthesis` — distilled framework, named model, doctrine document
- `decision` — strategic decision with reasoning, context, and alternatives considered

Track: How to create, package, distribute, monetize, and sustain value in digital form.

STEP 3 — DISTRIBUTION LITERACY
  Why most brilliant things are not discovered
  How attention works in digital environments
  Channel strategy — how to choose where your signal will be heard
  Positioning logic — how to own a distinct mental territory

STEP 4 — EXECUTION ARCHITECTURE
  How to build systems that produce value reliably
  SOPs, workflows, asset pipelines
  The difference between doing and systematizing
  How to build leverage — when to remain 1→1, when to build 1→many

  Mastery proof: can document any process such that a different person achieves the same output

STEP 5 — MASTERY AND SCALING
  How competence compounds
  How systems compound (vs. skills which plateau)
  What is a defensible position in a market?
